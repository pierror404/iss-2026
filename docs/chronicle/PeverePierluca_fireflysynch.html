ConwayLife Sprint2

Introduction

Goal: avere un sistema che a seconda della distanza rilevata da un sonar faccia lampeggiare 3 lucciole in modo sincronizzato o non sincronizzato

Requirements

Il sistema deve soddisfare i seguenti requisiti:


    
    
  • R1: Ii sistema deve essere composto da 3 lucciole.
    • 
    
  • R2: ogni lucciola lampeggia inizialmente con una frequenza propria costante.
    • 
    
  • R3: il sistema è dotato di un sonar che misura la distanza di un ostacolo.
    • 
    
  • R4: quando la distanza rilevata è minore di una soglia DMIN, le lucciole devono
        lampeggiare in modo sincronizzato.
    • 
    
  • R5: quando la distanza torna maggiore o uguale a DMIN, le lucciole devono
        tornare a lampeggiare in modo non sincronizzato.
    • 
    
  • R6: il lampeggio delle lucciole deve essere visibile tramite una griglia.
    •

Requirement analysis

Il sistema è composto dai seguenti componenti:


    
    
  • 3 QActor lucciola, che gestiscono il lampeggio
        ON/OFF ed inviano il proprio stato ad una griglia.
    • 
    
  • Un QActor orchestratore, che riceve i dati dal sonar e decide quando attivare
        o disattivare la sincronizzazione.
    • 
    
  • Un QActor sonar (sonarmock), che simula la rilevazione della distanza
        di un ostacolo.
    • 
    
  • Un QActor (esterno) griddisplay per la visualizzazione della griglia.
    • 


Gli eventi presenti nel sistema devono essere 3:
    

  • Uno inviato dal sonar all'orchestratore che comunica la distanza
    • 

  • Uno, inviato se la distanza rilevata è minore di DMIN e quindi le lucciole si devono sincronizzare
    • 

  • Uno da inviare per far tornare a lampeggiare le lucciole a frequenze indipendenti
    • 




Gli eventi (per il codice completo fare riferimento qui: 

Event sync         : sync(TIMER) 
Event sonardata    : distance(D) 
Event unsync       : unsync(CMD)

Problem analysis

Il comportamento delle lucciole rimane modellato come una macchina a stati a due stati (oltre a quelli per gestire gli eventi):
  • turnOn
  • turnOff
Ogni lucciola utilizza un timer locale per alternare lo stato ON/OFF. La durata del timer può essere modificata dinamicamente per ottenere la sincronizzazione. Il sonar emette periodicamente un evento sonardata : distance(D). L’orchestratore, alla ricezione di tale evento, confronta la distanza con una soglia DMIN:
  • se la distanza è minore di DMIN e il sistema non è sincronizzato, l’orchestratore invia un evento sync a tutte le lucciole;
  • se la distanza è maggiore o uguale a DMIN e il sistema risulta sincronizzato, l’orchestratore invia un evento unsync per ripristinare il comportamento casuale.
