DDR system

Introduction

Goal: Costruire un sistema software che comanda un Differential Drive Robot (DDR) in modo che, partendo dalla posizione iniziale HOME, il robot si sposti lungo il perimetro di una stanza rettangolare vuota, tornando alla posizione HOME.

Requirements

Costruire un sistema software che comanda un Differential Drive Robot (DDR) in modo che, partendo dalla posizione iniziale HOME, il robot si sposti lungo il perimetro di una stanza rettangolare vuota.
Il DDR si deve fermare quando tornato in HOME.

Il DDR è anche sensibile ai dati rilevati da un Radar. Quando il Radar rileva un 'intruso' a una distanza minore di una distanza prefissata ``DMIN``:
  • se il DDR si sta muovendo , il DDR si ferma fino a che il Radar non rileva più un 'intruso' così vicino
  • se il DDR è fermo in HOME, il DDR ruota su sè stesso fino a che un 'intruso' così vicino scompare

Requirement analysis

Il sistema software preso in esame è composto dalle seguenti entità:
  • HOME: Nei requisiti il termine HOME identifica sia la posizione iniziale del DDR all’interno della stanza sia la posizione finale del percorso lungo il perimetro della stanza. Per formalizzare tale concetto è necessario introdurre i seguenti concetti:
    • stanza: viene modellata come un rettangolo appartenente a un piano euclideo bidimensionale, caratterizzato da una larghezza W e da un’altezza H, entrambe espresse in metri.
    • posizione: ogni posizione all’interno della stanza può essere identificata mediante una coppia di coordinate (x,y).
    Tuttavia, il robot non può essere considerato un punto materiale, poiché occupa una porzione di spazio. Si assume pertanto che il DDR abbia un ingombro quadrato di lato R. Di conseguenza, la stanza può essere discretizzata in unità robotiche di dimensione RxR, ottenendo una rappresentazione a griglia.
    In tale modello, ogni cella della griglia rappresenta una posizione valida per il robot. La posizione HOME viene quindi definita come una cella predefinita della griglia, sufficientemente ampia da contenere interamente il DDR, che rappresenta sia il punto iniziale sia il punto finale del percorso operativo.
    Quanto detto è formalizzabile con il seguente codice Kotlin (classi Posizione e Stanza): 
    	data class Stanza(val W: Double, val H: Double, val R: Double) { // H e W sono in unità robotiche
		val larghezzaMetri: Double get() = W * R
	    	val altezzaMetri: Double get() = H * R
	    	val home: Posizione = Posizione(0, 0)
	}
		
    	data class Posizione(val x: Int, val y: Int)
  • Radar: è un’entità proattiva incaricata di monitorare periodicamente l’area circostante il DDR, rilevando l’eventuale presenza di ostacoli o “intrusi” posti a una distanza inferiore a una soglia minima DMIN.
    La proprietà fondamentale del Radar consiste nel fornire al sistema informazioni relative alla distanza rilevata D, espressa in metri. Poiché il sistema viene sviluppato utilizzando il linguaggio QAK, il Radar viene formalizzato come un attore.
    È inoltre necessario definire il meccanismo attraverso cui tali informazioni vengono rese disponibili agli altri componenti del sistema, così da consentire al DDR di modificare il proprio comportamento in funzione dei dati acquisiti. A tal proposito si considerano due possibili approcci:
    1. il Radar comunica autonomamente la distanza rilevata tramite un evento, senza conoscere preventivamente il destinatario dell’informazione. In questo caso il Radar viene formalizzato come il seguente attore: 
      System ddrsystem
Event radar : distance(D)
Context ctxddrsys	ip [ host="localhost" 	port=8010 ]
QActor radar_event context ctxddrsys {
	[# 
		var D = 0
		var Timer = 1000L
	 #]
	State s0 initial {
		println("$name | starting") color green
	}
	Goto rileva_distanza
	
	State rileva_distanza{
		
		emit radar : distance(D)
	}
	Transition t0 
		whenTimeVar Timer -> rileva_distanza
}
    2. in alternativa, il Radar fornisce la distanza rilevata soltanto in risposta a una richiesta esplicita proveniente dal sistema. In questo caso il Radar viene formalizzato come il seguente attore: 
      System ddrsystem
Request getDistance : getDistance(D) "richiesta del sistema"
Reply   radar_repl : distance(D) for getDistance
Context ctxddrsys	ip [ host="localhost" 	port=8010 ]
QActor radar_req_rep context ctxddrsys {
	[# 
		var D = 0
		var Timer = 1000L
	 #]
	State s0 initial {
		println("$name | starting") color green
	}
	Goto rileva_distanza
	
	State rileva_distanza{
		
	}
	Transition t0 
		whenTimeVar Timer -> rileva_distanza
		whenRequest getDistance -> handleRequest
   
	State handleRequest{
 		onMsg( getDistance : getDistance(D) ){
 			replyTo getDistance with radar_repl : distance( $D )  
 		} 
	}  
	Transition t0 
		whenTimeVar Timer -> rileva_distanza
}
  • DDR: è un’entità reattiva, priva di capacità decisionali autonome: il suo compito consiste esclusivamente nell’eseguire i comandi ricevuti, comportandosi quindi come un semplice attuatore. Questa modellazione deriva direttamente dai requisiti del committente, il quale richiede la realizzazione di un sistema di controllo del robot e dispone già dell’implementazione del DDR, fornendone il relativo codice sorgente.
  • Controller: rappresenta il componente responsabile della logica di controllo del sistema e coordina il comportamento del DDR secondo le specifiche fornite dal committente.
    In condizioni normali, il controller guida il DDR lungo il perimetro della stanza, con partenza e arrivo nella posizione HOME.
    Il controller gestisce inoltre l’interazione con il Radar, sia ricevendo eventi sia richiedendo esplicitamente la distanza rilevata, a seconda del modello adottato. Quando il Radar rileva una distanza D inferiore alla soglia DMIN, il sistema entra in stato di allarme:
    • se il DDR è in movimento, viene arrestato;
    • se il DDR è fermo in HOME, viene attivata una rotazione sul posto.
    Il comportamento di allarme continua finché l’intruso rimane nella zona critica. Una volta ristabilite condizioni sicure, il controller riprende il comportamento precedentemente interrotto.
    Per formalizzare tale comportamento viene definito il seguente attore:
